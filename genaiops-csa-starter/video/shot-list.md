# Shot List and Visual Plan

## Visual language

Use a terminal-inspired rbcloud style: dark background, command prompts, clean architecture cards, lifecycle diagrams, and minimal animation.

## Scenes

| Scene | Visual |
|---|---|
| Hook | Prompt window: "Demo works" followed by production questions appearing around it. |
| Define GenAIOps | Six-stage lifecycle diagram. |
| Build | Cards for prompts, RAG, agents, tools, orchestration. |
| Evaluate | Test dataset flowing into quality gates. |
| Deploy | CI/CD path to app endpoint behind gateway. |
| Monitor | Split screen: system health and answer health. |
| Govern | Identity, RBAC, audit, responsible AI controls. |
| Optimise | Token usage, model routing, quotas, cost per outcome. |
| Reference architecture | App, APIM, agent/orchestrator, model, retrieval, safety, telemetry. |
| Close | Pilot checklist and call to download the guide. |

## Screen capture ideas

Use generated diagrams rather than private tenant screenshots. If showing Azure portals, use public docs pages or a demo tenant with no customer data.

## Assets to create

- Lifecycle diagram.
- Reference architecture diagram.
- Evaluation quality gate diagram.
- Monitoring split diagram.
- Pilot checklist slide.
