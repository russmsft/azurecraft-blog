# Public Content Governance Checklist

Use this checklist before publishing material on rbcloud.co.uk.

## Confidentiality review

- [ ] No customer names.
- [ ] No customer architecture diagrams.
- [ ] No private workshop agenda, notes, or speaker notes copied verbatim.
- [ ] No internal-only product claims or roadmap references.
- [ ] No commercial or account strategy details.
- [ ] No screenshots from tenant-specific portals.
- [ ] No private email, Teams, calendar, or document content.
- [ ] No security-sensitive implementation details.

## Public suitability

- [ ] Uses generic examples only.
- [ ] Explains concepts from public knowledge and personal interpretation.
- [ ] Makes clear that examples are illustrative.
- [ ] Avoids implying official Microsoft guidance unless linking to public docs.
- [ ] Includes a personal-project disclaimer.
- [ ] Includes AI-generated voice disclosure if synthetic narration is used.

## Quality review

- [ ] Clear audience and learning outcomes.
- [ ] Plain-English definitions.
- [ ] Practical architecture patterns.
- [ ] Downloadable guide and crib sheet.
- [ ] Accessible headings and link text.
- [ ] No broken links.
- [ ] Mobile-friendly layout.
- [ ] SEO title and description included.
